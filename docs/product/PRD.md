# 产品需求文档

## 1. 项目名称

动态数据脱敏系统。

## 2. 项目定位

本项目是数据库系统及安全课程的期末作业，面向高校学生信息保护场景，建设一个具备前后端展示能力的动态数据脱敏系统。

系统核心不是简单地把字段替换成星号，而是围绕数据库安全主题展示完整链路：

- 用户登录与身份识别。
- RBAC 角色权限控制。
- 敏感字段元数据管理。
- 基于角色的字段级脱敏规则配置。
- 学生信息与成绩数据的安全查询。
- 访问审计、规则变更审计和异常访问检测。
- 视图、函数、存储过程、触发器、索引等数据库知识点展示。

## 4. 建设目标

系统需要达到以下目标：

- 支持不同角色登录系统，并根据角色加载菜单权限、接口权限和数据权限。
- 支持学生基础信息和学生成绩数据查询。
- 支持对姓名、手机号、邮箱、身份证、地址、出生日期、家庭收入、银行卡号、成绩等敏感字段进行动态脱敏。
- 支持安全管理员维护敏感字段、脱敏方式、脱敏策略和角色规则分配。
- 支持审计员查看访问日志、规则变更日志和异常访问记录。
- 支持数据分析员使用脱敏或泛化后的数据生成统计报表。
- 支持课程答辩演示“同一数据、不同角色、不同显示结果”。

## 5. 用户角色

| 角色编码 | 角色名称 | 核心定位 |
|---|---|---|
| `SUPER_ADMIN` | 超级管理员 | 拥有全部权限，用于系统最高管理 |
| `SYSTEM_ADMIN` | 系统管理员 | 管理用户、角色、权限，不接触业务数据 |
| `DATA_ADMIN` | 数据管理员 | 管理学生数据资源、字段和数据质量 |
| `SECURITY_ADMIN` | 安全管理员 | 管理敏感字段、脱敏规则和安全策略 |
| `TEACHER` | 教师/教务人员 | 查询教学和教务相关学生信息 |
| `ANALYST` | 数据分析师 | 使用脱敏或泛化数据做统计分析 |
| `AUDITOR` | 安全审计员 | 查看访问日志、规则变更日志和异常访问 |
| `NORMAL` | 普通用户 | 低权限用户，只能查看高度脱敏数据 |

角色边界原则：

- 数据管理员管数据，不直接配置脱敏策略。
- 安全管理员配规则，不直接查询学生业务数据。
- 数据分析师用数据，不查看原始敏感数据。
- 安全审计员查行为，不修改业务数据和脱敏规则。

## 6. 权限模型

系统采用 RBAC 权限模型，并将权限分为三类：

| 权限类型 | 作用 | 示例 |
|---|---|---|
| `MENU` | 控制前端页面是否可见 | `MENU:STUDENT_QUERY` |
| `API` | 控制后端接口是否可调用 | `API:STUDENT_QUERY` |
| `DATA` | 控制数据访问粒度和安全能力 | `DATA:VIEW_RAW`、`DATA:VIEW_MASKED` |

核心判断逻辑：

```text
能否看到页面：判断 MENU 权限
能否调用接口：判断 API 权限
能否查看原始数据：判断 DATA:VIEW_RAW
能否查看脱敏数据：判断 DATA:VIEW_MASKED
能否配置脱敏规则：判断 DATA:CONFIG_MASKING
能否查看审计日志：判断 DATA:AUDIT_ACCESS
```

## 7. 功能模块

### 7.1 登录认证模块

用户输入账号密码登录。后端校验密码哈希，加载用户角色和权限，生成登录令牌，并返回前端渲染菜单所需信息。

登录成功后应返回：

- 用户 ID。
- 用户名。
- 真实姓名。
- 角色列表。
- 权限编码列表。
- 令牌信息。

### 7.2 用户角色权限管理模块

面向 `SUPER_ADMIN` 和 `SYSTEM_ADMIN`。

功能包括：

- 用户列表、新增、修改、禁用。
- 角色列表、新增、修改、禁用。
- 权限列表查看。
- 用户角色分配。
- 角色权限分配。

对应数据表：

- `sys_user`
- `sys_role`
- `sys_permission`
- `sys_user_role`
- `sys_role_permission`

### 7.3 学生信息管理模块

面向 `SUPER_ADMIN` 和 `DATA_ADMIN`。

功能包括：

- 学生基础信息新增、修改、删除。
- 学生成绩新增、修改、删除。
- 学生数据质量维护。

注意：课程项目中建议 `DATA_ADMIN` 默认查看脱敏数据，不授予无限制原始数据查看能力。

### 7.4 学生信息查询模块

这是核心演示模块。

用户通过条件查询学生信息，后端根据当前角色和权限决定返回原始数据或脱敏数据。

推荐查询方式：

- 普通条件查询为主，支持学院、专业、年级、班级、姓名、学号筛选。
- SQL 查询框可作为高级演示功能，但必须进行安全限制。

查询流程：

```text
用户进入学生查询页面
  -> 前端根据 MENU 权限显示页面
  -> 用户提交查询条件
  -> 后端校验 API:STUDENT_QUERY
  -> 后端校验 DATA 权限
  -> 查询 student_info / student_score 原始数据
  -> 判断是否拥有 DATA:VIEW_RAW
  -> 如有原始数据权限，返回原始数据
  -> 如只有脱敏权限，加载角色脱敏规则
  -> 对敏感字段逐字段脱敏
  -> 返回结果
  -> 写入访问审计日志
```

### 7.5 敏感字段管理模块

面向 `SUPER_ADMIN`、`SECURITY_ADMIN`，`DATA_ADMIN` 可查看。

功能包括：

- 查看敏感字段列表。
- 新增或修改敏感字段元数据。
- 启用或禁用字段保护。
- 维护敏感类型、敏感等级和识别方式。
- 可选：按字段名或正则扫描潜在敏感字段。

对应核心表：`sensitive_field`。

### 7.6 脱敏规则管理模块

面向 `SUPER_ADMIN` 和 `SECURITY_ADMIN`。

功能包括：

- 查看脱敏方式字典。
- 查看字段级脱敏策略。
- 新增、修改、删除、启用、禁用脱敏策略。
- 将脱敏策略分配给角色。
- 测试脱敏效果。
- 查看规则变更日志。

对应核心表：

- `masking_type_dict`
- `masking_policy`
- `masking_rule_assignment`
- `rule_change_log`

### 7.7 数据分析报表模块

面向 `SUPER_ADMIN` 和 `ANALYST`。

功能包括：

- 按学院、专业、年级统计人数。
- GPA 分布分析。
- 年龄分布分析。
- 家庭收入区间分析。
- 成绩区间分析。
- 导出脱敏报表。

数据分析员不得查看身份证、银行卡、完整手机号、完整地址等原始敏感信息。

### 7.8 安全审计模块

面向 `SUPER_ADMIN` 和 `AUDITOR`，`SECURITY_ADMIN` 可查看规则变更日志。

功能包括：

- 查看访问日志。
- 查看敏感字段访问记录。
- 查看脱敏命中情况。
- 查看规则变更日志。
- 查看异常访问记录。
- 导出审计报告。

对应核心表：

- `access_log`
- `rule_change_log`
- `abnormal_access`

## 8. 脱敏方式

系统计划支持以下脱敏方式：

| 脱敏方式 | 说明 | 示例 |
|---|---|---|
| `NO_MASK` | 不脱敏 | `张三` |
| `FULL_MASK` | 全部遮蔽 | `***********` |
| `KEEP_PREFIX` | 保留前缀 | `张*` |
| `KEEP_SUFFIX` | 保留后缀 | `********0123` |
| `KEEP_PREFIX_SUFFIX` | 保留前后 | `138****5678` |
| `EMAIL_MASK` | 邮箱脱敏 | `z****@school.edu.cn` |
| `ADDRESS_LEVEL` | 地址层级脱敏 | `四川省成都市****` |
| `GENERALIZATION` | 区间泛化 | `80000-90000` |
| `HASH_MASK` | 哈希脱敏 | `a8f3c9d1` |
| `KEEP_YEAR` | 日期只保留年份 | `2003-**-**` |
| `NOISE` | 数值加噪 | `原值上下浮动区间` |

## 10. 数据库展示重点

系统应在课程答辩中明确展示以下数据库知识点：

- 14 张核心表的分层设计。
- RBAC 五张权限表。
- 业务表 `student_info` 与 `student_score` 拆分，体现第三范式。
- 敏感字段、脱敏方式、脱敏策略、规则分配四表解耦。
- 审计日志与异常访问记录。
- 外键约束与级联策略。
- 常用查询索引。
- 统一脱敏函数 `FN_APPLY_MASK`。
- 动态脱敏查询存储过程。
- 异常检测存储过程。
- 策略变更触发器。
- 脱敏配置视图 `V_MASKING_CONFIG`。

## 12. 答辩演示流程

推荐演示顺序：

1. 超级管理员登录，展示完整菜单和系统概览。
2. 安全管理员登录，配置敏感字段和脱敏规则。
3. 教师登录，查询学生信息，展示轻度脱敏结果。
4. 数据分析师登录，查看泛化后的统计数据和报表。
5. 普通用户登录，查看高度脱敏结果。
6. 安全审计员登录，查看访问日志、规则变更日志和异常访问记录。

## 13. 非目标范围

当前阶段不做以下内容：

- 不继续沿用旧 C++ TCP 代理实现。
- 不做真实生产级数据防泄漏系统。
- 不做复杂 SQL 自由执行能力作为主流程。
- 不做过度复杂的多租户、分布式部署和大数据平台集成。
